## Answer Template for Common Questions

### "Tell me about your project"
**Answer:** 
"I developed an algorithmic trading platform with a Python FastAPI backend, React/Next.js frontend, and PostgreSQL database. The system lets users create trading strategies, backtest them against historical data using Backtrader, and simulate live trading. It's 75% complete with a fully functional backend API, authentication system, and trading engine. I'm currently working on the frontend dashboard and risk management features."

### "What was the most challenging part?"
**Answer:**
"Implementing accurate financial calculations and integrating them with the Backtrader engine was challenging. Specifically, correctly calculating Sharpe Ratio, Max Drawdown, and managing strategy parameters. I solved this by studying financial literature and thoroughly understanding Backtrader's documentation and parameter system."

### "How does your authentication work?"
**Answer:**
"Users register with email/password. Passwords are hashed using bcrypt before storage. On login, JWT tokens are generated by the backend and stored in localStorage on the frontend. The axios client automatically includes this token in API request headers, and we have interceptors to handle token expiration and auto-redirect to login on 401 errors."

### "What technologies did you use and why?"
**Answer:**
"FastAPI for the backend because it's fast, async, and auto-generates API documentation. Next.js for the frontend because it provides great React development experience with server-side rendering. PostgreSQL for reliability and ACID compliance. Docker for consistent containerization. Backtrader for backtesting because it's the industry standard. And yfinance for free, no-API-key market data."

### "What would you do differently?"
**Answer:**
"I'd implement testing earlier - I have documentation-ready pytest structure but minimal tests. I'd also plan more time for frontend development, as it's taking longer than backend. I'd start with a component library to speed up UI development. And I'd add WebSocket support from the beginning for real-time updates."

## Quick Command Reference

```bash
# Start everything
docker-compose up --build

# Check specific service
docker-compose ps

# View logs
docker-compose logs backend
docker-compose logs frontend

# Stop everything
docker-compose down

# Clean up
docker system prune

# Manual local run
cd backend && pip install -r requirements.txt
uvicorn app.main:app --reload

cd frontend && npm install && npm run dev
```
